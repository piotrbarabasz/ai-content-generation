*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- Modular workflow first: The plan MUST show how the feature fits the shared workflow engine and how modules can be enabled, disabled or swapped.
- Module contracts: The plan MUST identify impacted modules and their required input/output/config contracts.
- Provider abstraction: The plan MUST name the provider interfaces and show how external systems are isolated.
- Artifact traceability: The plan MUST include artifact persistence, metadata and replay expectations.
- Review and approval: The plan MUST mark any script, scene plan, QA, voiceover, caption or render checkpoints that require human review.
- MVP scope discipline: The plan MUST keep the feature within the MVP boundary for short video and long-form script/voiceover workflows.
- Narrative/render separation: The plan MUST preserve a distinction between narrative segments and render scenes.
- Testability: The plan MUST include deterministic tests for contracts, retries, toggles and artifact outputs.
- No hardcoded local paths: The plan MUST use configuration and artifact store abstractions instead of absolute paths.
